# 指令参考手册

本文档提供 ARTA 所有指令的详细说明。

## 指令格式

ARTA 指令采用以下格式：

```
/ARTA-<模块>-<操作> [参数]
```

---

## 📁 项目管理指令

### `/ARTA-init`

**说明**：初始化新项目

**用法**：
```
/ARTA-init
```

**执行流程**：
1. 清除当前项目配置
2. 进入项目接入引导流程
3. 依次询问项目类型、代码来源等信息

**示例**：
```
用户输入: /ARTA-init
Agent: 🚀 开始初始化新项目...
       请问您的项目是：
       A. 🆕 全新项目
       B. 📁 已有项目（已有自动化测试）
       C. 🔄 已有项目（无自动化测试）
```

---

### `/ARTA-project-set`

**说明**：设置本地项目路径

**用法**：
```
/ARTA-project-set <项目路径>
```

**参数**：
- `<项目路径>`：本地项目的绝对路径

**示例**：
```
Windows: /ARTA-project-set C:\Users\Dev\projects\my-api
Linux/Mac: /ARTA-project-set /home/dev/projects/my-api
```

**执行流程**：
1. 验证路径有效性
2. 扫描项目结构
3. 识别框架类型
4. 提取 API 路由
5. 生成 API 概况

---

### `/ARTA-openapi-set`

**说明**：设置 OpenAPI/Swagger 规范来源

**用法**：
```
/ARTA-openapi-set <URL或路径>
```

**参数**：
- `<URL或路径>`：OpenAPI 文件的 URL 或本地路径

**示例**：
```
URL: /ARTA-openapi-set https://api.example.com/openapi.json
本地文件: /ARTA-openapi-set ./docs/swagger.yaml
```

**支持格式**：
- OpenAPI 3.0 / 3.1 (JSON/YAML)
- Swagger 2.0 (JSON/YAML)

### `/ARTA-project-analyze`

**说明**：重新分析项目，更新 API 概况

**用法**：
```
/ARTA-project-analyze
```

**执行流程**：

### `/ARTA-api-add`

**说明**：手动添加 API

**用法**：
```
/ARTA-api-add <方法> <路径> <描述> [模块]
```

**参数**：
- `<方法>`：HTTP 方法 (GET/POST/PUT/PATCH/DELETE)
- `<路径>`：API 路径
- `<描述>`：API 功能描述
- `[模块]`：可选，所属模块

**示例**：
```
/ARTA-api-add POST /api/auth/logout 用户登出 认证
```

---

### `/ARTA-api-edit`

**说明**：编辑指定 API 信息

**用法**：
```
/ARTA-api-edit <序号>
```

**参数**：
- `<序号>`：API 在列表中的序号

**示例**：
```
用户输入: /ARTA-api-edit 3
Agent: 当前 API 信息：
       路径: /api/users
       方法: GET
       描述: 获取用户列表
       模块: 用户
       
       请选择要修改的项目：
       1. 路径
       2. 方法
       3. 描述
       4. 模块
       
用户输入: 3
Agent: 请输入新的描述：
用户输入: 获取所有用户信息，支持分页和筛选
Agent: ✅ API #3 描述已更新
```

---

/ARTA-api-delete <序号>

## 📄 输出指令

### `/ARTA-generate-cases`

**说明**：基于业务链路生成测试用例

**用法**：
```
/ARTA-generate-cases [链路序号]
```

**参数**：
- `[链路序号]`：可选，指定链路；不指定则生成所有链路的用例

**输出**：
- 生成 Markdown 格式的测试用例文档
- 包含测试步骤、预期结果、测试数据

### 🤖 Agent 协调指令

### `/ARTA-coord-analyze`

**说明**：协调多 Agent 分析指定模块

**用法**：
```
/ARTA-coord-analyze <模块名>
```

**参数**：
- `<模块名>`：要分析的模块名称

**执行流程**：
1. 调用 arta-analyzer 分析模块 API
2. 调用 arta-pattern-learner 加载相关模式
3. 合成分析结果

**示例**：
```
用户输入: /ARTA-coord-analyze 订单

┌────────────────────────────────────────────────────────────┐
│ 任务: 订单模块分析                                          │
├────────────────────────────────────────────────────────────┤

│   [arta-analyzer] 分析订单模块 API                         │
│   [arta-pattern-learner] 加载订单相关模式                  │
├────────────────────────────────────────────────────────────┤
│ 结果:                                                       │
│   - 识别 API: 8 个                                          │
│   - 加载模式: 3 个                                          │
│   - 分析完成                                                │
└────────────────────────────────────────────────────────────┘
```
