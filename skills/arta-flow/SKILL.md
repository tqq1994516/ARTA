# ARTA 业务链路记录模块

## 角色定义

你专注于业务链路记录，帮助用户：
- 记录完整的业务流程
- 关联 API 调用序列
- 定义测试数据和断言
- 生成业务流程图

## 触发指令

| 指令 | 说明 |
|------|------|
| `/ARTA-flow-list [状态]` | 查看所有链路 |
| `/ARTA-flow-add [名称]` | 添加业务链路 |
| `/ARTA-flow-edit <序号>` | 编辑链路 |
| `/ARTA-flow-delete <序号>` | 删除链路 |
| `/ARTA-flow-view <序号>` | 查看链路详情 |

## 学习机制集成

### 链路完成时自动学习

当业务链路状态变为 `completed` 时，自动触发学习：

```
业务链路完成
    ↓
读取学习配置 (assets/configs/learning_config.json)
    ↓
提取模式:
├── API 序列模式
│   - 提取调用顺序: login → get_products → create_order
│   - 记录模块归属
│   - 计算置信度
│
├── 数据策略模式
│   - 识别数据传递: token, userId
│   - 记录数据来源
│   - 标注变量依赖
│
└── 断言模式
    - 提取常见断言组合
    - 记录 API 方法关联
    - 更新断言模板库
    ↓
更新模式存储 (assets/data/patterns.json)
    ↓
更新学习统计
```

### 模式提取示例

```
链路: 用户下单流程
提取的模式:

1. API 序列模式:
   - 序列: POST /login → GET /products → POST /orders
   - 频次: 首次记录 (frequency: 1)
   - 模块: 订单
   - 置信度: 基础值 0.5

2. 数据策略模式:
   - token 传递: login.response.token → Authorization
   - 适用场景: 需要认证的 API

3. 断言模板:
   - 登录成功断言: status=200 + token exists
   - 创建订单断言: status=201 + orderId exists
```

## 流程图生成

业务链路保存后，自动调用脚本生成流程图：

```python
python scripts/generate_flow_diagram.py --flow-id <链路ID>
```

流程图配置见 `assets/configs/flow_diagram_config.json`

## 链路状态

| 状态 | 说明 |
|------|------|
| draft | 草稿 - 正在编辑中 |
| completed | 完成 - 可用于生成测试用例 |
| deprecated | 已弃用 - 业务已下线 |

## 优先级说明

| 优先级 | 说明 | 示例 |
|--------|------|------|
| P0 | 核心流程 | 登录、下单、支付 |
| P1 | 重要流程 | 搜索、评论、收藏 |
| P2 | 一般流程 | 个人设置、通知 |
| P3 | 边缘流程 | 帮助、反馈 |
