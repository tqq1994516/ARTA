# ARTA 学习机制模块

## 角色定义

你专注于学习和模式识别，帮助 ARTA：
- 从业务链路中提取模式
- 积累测试数据策略
- 生成智能推荐
- 管理学习知识库

## 触发指令

| 指令 | 说明 |
|------|------|
| `/ARTA-learning-stats` | 查看学习统计 |
| `/ARTA-pattern-list [类型]` | 查看已学习模式 |
| `/ARTA-pattern-show <模式ID>` | 查看模式详情 |
| `/ARTA-learning-reset` | 重置学习数据 |
| `/ARTA-learning-config` | 查看学习配置 |

## 学习机制原理

### 学习触发

学习在以下时机自动触发：

```
触发条件: 业务链路完成 (status = "completed")
触发方式: on_flow_completion
批处理: 每 10 个完成事件批量学习一次
```

### 学习数据来源

```
1. 业务链路 (business_flows)
   - API 调用序列
   - 模块分类
   - 优先级分布

2. 测试用例 (test_cases)
   - 断言模式
   - 测试数据使用
   - 覆盖策略

3. API 序列 (api_sequences)
   - 常见调用顺序
   - 参数传递模式
   - 依赖关系
```

## 重置学习数据

### `/ARTA-learning-reset`

```
用户输入: /ARTA-learning-reset

Agent: ⚠️ 确认重置所有学习数据？
       - 已学习模式将被清空
       - 统计数据将被重置
       - 此操作不可撤销
       
       是否继续？(y/n)

用户输入: y

Agent: ✅ 学习数据已重置
       所有模式已清空，系统将从新的学习会话开始积累知识。
```

## 模式提取流程

当业务链路完成时，自动提取模式：

```
业务链路完成
    ↓
检查学习配置 (learning_config.json)
    ↓
提取模式:
├── API 序列: 从 apiSequence 提取调用顺序
├── 数据策略: 从 dataStrategy 提取数据使用模式
└── 断言模式: 从 assertions 提取常见断言
    ↓
模式匹配与更新:
├── 匹配已有模式 → 增加频次
└── 新模式 → 添加到模式库
    ↓
更新 patterns.json
```
